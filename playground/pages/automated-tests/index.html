How does it work

  1. Initially, the repository webcardinal-tests is cloned inside .dev/webcardinal-tests.
  2. Then by octopus runner, tests are moved in the development root of the component library, in test directory.
    (e.g .dev/webcardinal/.webcardinal/components/webcardinal-core/test).
  3. At that location jest and puppeteer will be installed.
    (currently are installed by octopus for webcardinal-core)

In a nutshell, a Chromium instance is used for E2E testing.


Related: StencilJS E2E testing

What is an WebCardinal Testing Environment

A test environment is a WebCardinal application designed specifically for testing.

An environment contains pages, templates, controllers, etc.

It should supplies auxiliary methods and helpers.

Currently only webcardinal-core offers a testing environment.

How to run current tests

From the root of this application run the following command:

npm run test # npm t
